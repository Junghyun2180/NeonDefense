00OVERVIEW

디자인 컨셉 & 톤

"지휘관이 작전실에서 홀로그램 맵을 내려다본다" — SF 사령부의 메타포를 차용한 UX 리디자인안입니다. 본 문서는 Holographic Command(이하 Holo) 시안의 최종 컬러·타이포·컴포넌트·레이아웃을 정의해, 게임 클라이언트에 그대로 이식할 수 있는 수준의 명세를 제공합니다. 컬러 톤은 1차 Holo의 시안 일색 → 크림슨 플라즈마 → Minimal Neon의 다크 뉴트럴 베이스를 흡수해 최종적으로 "미니멀 베이스 + 크림슨/앰버 액센트" 형태로 안착했습니다.

핵심 원칙

  • Hairline-first 1px 보더 · gradient 최소
  • Mono = data, Sans = chrome
  • 액센트는 텍스트·아이콘에만 패널 면적 X
  • 색은 가능한 적게, 의미가 있을 때만

비주얼 정체성

  • 레티클 코너 14px L-shape
  • 모노 이브로우 letter-spacing 3px
  • 가로 게이지 바이탈 스탯
  • 플랫 다크 #08080a 베이스

피해야 할 것

  • 오버레이 그라디언트 패널
  • 2px 이상의 보더
  • 색상 6개 이상 한 화면 동시 사용
  • 둥근 모서리 (radius ≥ 6px)

07INTERACTION & MOTION

인터랙션 · 모션

모션은 화려함이 아니라 정보 전달의 보조. 페이지 전환은 fade only, 프레스/호버는 1프레임 짜리 보더 색 변경 위주, 데이터 변화에만 짧은 펄스.

  • 호버 보더 8% → 16% / text dim → text. 100ms ease-out
  • 탭 전환 underline 2px slide. 180ms cubic-bezier(0.4, 0, 0.2, 1)
  • 웨이브 시작 크림슨 라인이 헤더 좌→우 1회 통과 (400ms)
  • 골드 변화 숫자 카운트업 250ms · 양수 +초록 · 음수 +빨강 1회 펄스
  • 타워 배치 레티클 4코너가 200ms 사이 안쪽으로 수렴 → 페이드 인
  • 적 처치 2px 화이트 플래시 → 80ms 페이드. 사운드와 동기화.
  • 금지 바운스 / 회전 / 스케일 1.x 이상 / 보라 글로우

  • 크로스해치는 CSS로 background-image

경로 화살표

  • 에셋 파일 없음 — 데이터·코드 생성

9.2 구현 노트 · 렌더 아키텍처

경로는 타일마다 화살표 PNG를 까는 방식이 아니라, 단일 SVG <polyline> 하나가 모든 타일 중심을 관통하는 방식. 코너 처리·dash 연결·길이 조절이 SVG의 자연스러운 동작으로 해결됩니다. 그리드/코어/사거리는 CSS로.

    linear-gradient(to right, rgba(255,255,255,0.04) 1px, transparent 1px),
    linear-gradient(to bottom, rgba(255,255,255,0.04) 1px, transparent 1px);
  background-size: 64px 64px;
}

/* 9.2.2 ・ 경로 렌더링 — SVG polyline 기반 (shared/game-scene.jsx 구현 참고) */
// 1. 데이터 모델 — 타일 좌표 배열 → 픽셀 중심 → polyline points 문자열
const PATH_TILES = [[0,1],[1,1],[2,1],[3,1],[3,2],[3,3], /* ... */];
const pathCenters = PATH_TILES.map(([c,r]) => [c*TILE+TILE/2, r*TILE+TILE/2]);
const pathPoints  = pathCenters.map(([x,y]) => `${x},${y}`).join(' ');

// 2. 레이어 B — 단일 polyline이 모든 타일 중심을 관통 (연결되어 보이는 핵심)
<polyline
  points={pathPoints}
  fill="none"
  stroke="var(--nd-el-water)"
  strokeWidth="2"
  strokeDasharray="6 6"
  strokeLinecap="round"          // 대시 끝이 둥그래서 이어짐
  strokeLinejoin="round"         // 코너에서 끝김 없이 돌아감
  opacity="0.6" />

// 3. 레이어 C — 4타일마다 화살촉 폴리곤, 로컬 prev/next로 각도 계산
{pathCenters.filter((_, i) => i > 0 && i < pathCenters.length-1 && i % 4 === 0)
  .map(([x,y], i) => {
    const idx  = pathCenters.findIndex(p => p[0]===x && p[1]===y);
    const prev = pathCenters[Math.max(0, idx-1)];
    const next = pathCenters[Math.min(pathCenters.length-1, idx+1)];
    const angle = Math.atan2(next[1]-prev[1], next[0]-prev[0]) * 180/Math.PI;
    return <g transform={`translate(${x},${y}) rotate(${angle})`}>
      <polygon points="-5,-4 5,0 -5,4" fill="var(--nd-el-water)" opacity="0.5" />
    </g>;
  })
}

/* 4. 흐르는 대시 애니메이션 — polyline의 stroke-dashoffset만 이동 */
@keyframes flow {
  to { stroke-dashoffset: -12; } // dasharray의 합(6+6)만큼 이동 = 완벽 루프
}
polyline.flow { animation: flow 1.2s linear infinite; }
