# [제안 2] 이벤트 웨이브 & 동적 맵 기믹

## TL;DR
웨이브마다 단조로운 "적 N마리 스폰" 대신, **이벤트 웨이브(8종)** 와 **맵 기믹 타일(4종)** 을 투입해
캠페인/런모드 후반부의 반복감을 깨고 매 웨이브가 독립된 "작은 퍼즐" 이 되도록 만든다.

## 문제 정의 (Why)
- 현재 캠페인 전 웨이브의 **구조가 동일**: 스폰 → 처치 → 다음. 마지막 웨이브(W5)만 보스로 차별화.
- `SPAWN_RULES` 는 보스 1건만 고정. 나머지는 `SPECIAL_ENEMY_CHANCE` 확률 블렌딩뿐이라 **"이번 웨이브는 뭐가 나올까"** 의 기대감이 없다.
- 맵은 단순한 경로 타일뿐이라 **지형 전략 선택지가 0**. 어디에 배치하든 시야/사거리만 고려하면 된다.
- S3~S6 스테이지의 체감 길이(5웨이브 × 6스테이지 = 30웨이브)에 비해 **의외성/이벤트성** 이 빈약. 런모드 반복 플레이 시 특히 단조롭다.
- Stage 1~2는 해금 페이스가 빠르지만, 일단 전 몬스터 풀이 열린 S5 이후에는 **새로운 자극이 없다.**

## 참고 사례 (역기획)

### Kingdom Rush — 웨이브 내러티브
- 각 웨이브에 고유 이름("Goblin Raid", "Troll Chief Arrives") 과 **전조 메시지** 가 붙는다. 웨이브 자체가 작은 에피소드.
- 종종 스폰 포인트 1개만 쏟아붓거나, 스폰 포인트를 교체해 **플레이어가 재배치를 고민하게** 만든다.
- 유저 반응: 반복 플레이에도 "웨이브 순서" 를 기억하게 되어 전략 학습곡선이 생긴다.
- 우리와 차이: 우리는 경로가 씨드 기반 랜덤이라 Kingdom Rush처럼 고정 스크립트는 못 씀. 대신 **웨이브 타입 풀에서 랜덤 선택** 방식이 적합.

### Plants vs Zombies — 맵 기믹 (수영장, 지붕, 안개)
- 맵마다 타일 종류가 달라 사용 가능 식물이 바뀐다. "어디에 어떤 타워" 라는 의사결정이 맵 자체에서 발생.
- 우리와 공통점: 그리드 기반 배치. 기믹 타일 추가가 기술적으로 자연스럽다.

### Infinitode 2 — 특수 웨이브 (Boss Wave, Flying Wave, Split Wave)
- 5웨이브마다 보스, 10웨이브마다 특수. 플레이어가 "이번엔 뭐가 올까" 하고 쿼 리 웨이브 의존적으로 빌드 조정.
- 우리 맥락: 6스테이지 × 5웨이브는 짧은 편이라, **스테이지마다 1~2 이벤트 웨이브 고정 슬롯** 으로 충분하다.

### Bloons TD 6 — MOAB-class, Camo Regrow
- 단순히 "강한 적" 뿐 아니라 "재생하는 위장 적" 처럼 **기존 타워 빌드 한계를 드러내는 특수 조건** 의 웨이브가 있다.
- 우리 맥락: 특정 속성 약화/강화 웨이브로 재현 가능.

## 제안 기능 (What)

### A. 이벤트 웨이브 (Event Wave) — 8종

스테이지 내 5웨이브 중 **W2, W4** 를 이벤트 웨이브 후보 슬롯으로 지정. 50% 확률로 이벤트 발동.
(W5는 보스 고정, W1/W3는 일반 유지 → 리듬감 보존)

| ID | 이름 | 조건 | 보상 |
|---|---|---|---|
| `rushWave` | 러시 웨이브 | `fast` 타입만 50% 증가한 수로 스폰 | 웨이브 골드 +30% |
| `tankWave` | 중장갑 웨이브 | `elite` 비율 80%, 수는 50% 감소 | 웨이브 골드 +30% |
| `swarmWave` | 군단 웨이브 | 적 수 2배, 개별 체력 ×0.5 | 웨이브 골드 +20% |
| `splitterWave` | 분열 침공 | 전원 `splitter` | 웨이브 골드 +40% |
| `jammerWave` | 전파 교란 | `jammer` 50% + 타워 사거리 -20% 이번 웨이브 한정 | 웨이브 골드 +50% |
| `fogWave` | 안개 웨이브 | 타워 사거리 표시 숨김 + 적 이동 속도 +15% | 웨이브 골드 +40% |
| `convoyWave` | 호송 웨이브 | `healer` 1기 + 일반 적 20기를 힐러가 보호 | 웨이브 골드 +50%, 힐러 처치 시 추가 드로우 1회 |
| `timeBombWave` | 시한 웨이브 | 45초 제한. 시간 초과 시 남은 적 전원 즉시 lives 감소 | 웨이브 골드 +50% |

### B. 맵 기믹 타일 (Map Gimmick) — 4종

스테이지별 시드 기반으로 **1~3개** 기믹 타일이 맵에 배치됨 (경로가 아닌 빈 타일 중 선택).

| ID | 이름 | 효과 | 연출 |
|---|---|---|---|
| `crystalNode` | 크리스탈 노드 | 타워가 이 타일에 배치되면 `damage +15%` + `range +10%` | 반짝이는 청록 타일 |
| `dampField` | 억제 필드 | 타워가 이 타일에 배치되면 `damage -10%` 그러나 이 타일 반경 80px 내 적 `받는 피해 +20%` (취약) | 흐릿한 보라 타일 |
| `speedRail` | 속도 레일 (경로 위) | 적이 이 타일을 지날 때 `speed ×1.5` (3초) | 노란 화살표 타일 |
| `choke` | 협곡 | 이 타일 반경 60px에 있는 적의 `받는 피해 +30%` (타워 미배치) | 붉은 균열 타일 |

**규칙**
- 맵 기믹은 스테이지 시작 시 **씨드 기반 1회 배치**, 리트라이 시에도 동일(런 시드로 재생성).
- 경로 위 기믹(`speedRail`)은 경로 타일만 선택. 경로 외 기믹(`crystalNode`, `dampField`, `choke`)은 빈 타일만.
- 런모드 ㅁ맵은 경로 비율이 높아 `speedRail` 가중치 ↑.
- 보스 러시는 기믹 비활성(단일 보스 전투 혼란 방지).

### 예시 수치

**스테이지별 기믹 개수**
| 스테이지 | 크리스탈 | 억제 | 레일 | 협곡 |
|---|---|---|---|---|
| S1 | 1 | 0 | 0 | 0 |
| S2 | 1 | 1 | 0 | 0 |
| S3 | 1 | 1 | 1 | 0 |
| S4~6 | 2 | 1 | 1 | 1 |

**이벤트 웨이브 활성 확률**
| 스테이지 | W2 이벤트 확률 | W4 이벤트 확률 |
|---|---|---|
| S1 | 0% (튜토리얼) | 30% |
| S2 | 30% | 40% |
| S3~6 | 40% | 60% |

## 구현 영역 (Where)

### 영향 받는 도메인/파일

**domain/config**
- `constants.js`
  - `EVENT_WAVE_TYPES` 신규 상수 (8종 정의: id/name/icon/modifier 함수)
  - `MAP_GIMMICK_TYPES` 신규 상수 (4종 정의: id/effect 함수/렌더 정보)
  - `EVENT_WAVE_CHANCE`, `GIMMICK_COUNT_PER_STAGE` 테이블
- `game-data.js` — 모드별 이벤트 웨이브 활성화 여부 플래그 추가 (`modes.campaign.eventWaves: true`, `modes.bossRush: false`)

**domain/enemy**
- `enemy-system.js`
  - `create()` 내 `eventWaveType` 주입 시 `healthMult`, 수, 타입 오버라이드
  - `determineType()` 에 이벤트 웨이브 분기 추가

**domain/combat**
- `game-engine.js`
  - 웨이브 시작 시 이벤트 웨이브 룰 평가 (`rollEventWave(stage, wave)`)
  - 맵 기믹 체크: 타워 배치 시 `gridX/gridY` 기반 기믹 효과 적용
  - 적 이동 처리에 `speedRail` 체크 추가

**domain/tower**
- `tower-system.js`
  - `placeOnGrid()` 호출 시 기믹 타일 보너스 반영 (`tower.gimmickBonus = {damageMult, rangeMult}`)

**infra**
- `run-mode.js` / `save-system.js` — 스테이지 시작 시 기믹 씨드 결정/복원

**hooks**
- `useGameLoop.jsx` — 웨이브 시작 시 이벤트 알림 표시 트리거
- `useGameState.jsx` — `currentEventWave`, `mapGimmicks[]` 상태 추가

**components**
- `GameMap.jsx` — 기믹 타일 렌더링
- `GameHeader.jsx` — 이벤트 웨이브 배너 표시 (상단 1.5초 페이드)
- `WaveAnnouncer.jsx` (신규) — 이벤트 웨이브 전용 큰 타이포 연출

### 새 파일 경로 제안
- `js/domain/combat/event-wave-system.js` (신규, 이벤트 웨이브 평가/적용 로직 분리)
- `js/domain/combat/map-gimmick-system.js` (신규, 기믹 타일 생성/효과 적용)

## 리스크 & 오픈 이슈

### 밸런스
- **이벤트 웨이브 난이도 스파이크**: `tankWave` 가 초반 스테이지에서 나오면 빌드가 없어 클리어 불가. → S1은 이벤트 비활성(확률 0% 또는 무해한 `swarmWave` 만 허용).
- **골드 인플레이션**: 이벤트 보상 +% 가 누적되면 T4 조기 달성 → **이벤트 확률 ≤ 50%** 유지 + `BalanceDesigner` 핸드오프로 전체 경제 재튜닝.
- **`fogWave` UX 리스크**: 타워 사거리 숨김 기믹이 모바일에서 학습 장벽. 최초 1회 튜토리얼.

### 기존 시스템 충돌
- `SPAWN_RULES` 와 `SPECIAL_ENEMY_CHANCE` 확률이 이벤트 웨이브 타입에 **덮어쓰기** 인지 **가산** 인지 규칙 명시 필요.
  → **덮어쓰기** (이벤트 웨이브가 활성이면 기존 `SPECIAL_ENEMY_CHANCE` 무시) 제안.
- 맵 기믹 타일과 **경로 생성 시드** 가 충돌하지 않도록 "경로 확정 → 기믹 배치" 순서 고정.
- `supportTowers` 버프 오라와 `crystalNode` 보너스가 중첩 시 캡(`SUPPORT_CAPS`) 적용 여부 → **별도 보너스 채널** 로 캡 밖에서 처리 제안.

### 토큰/성능
- 16×12 = 192 타일. 각 타일에 기믹 효과 체크하면 프레임당 오버헤드.
  → 기믹 타일은 **Map<gridKey, gimmick>** 로 O(1) 조회. 기믹이 없으면 체크 자체 스킵.
- `EVENT_WAVE_TYPES` 는 데이터 + 콜백이라 `constants.js` 만으로는 부족할 수 있음 → `domain/combat/event-wave-system.js` 로 분리 권장.

### UX 혼선
- 이벤트 웨이브 진입 시 **일시정지 1초** → 플레이어가 준비할 시간 확보 (런모드는 자동 시작이라 특히 중요).
- 기믹 타일은 **hover/touch 툴팁** 필수. 모바일은 long-press 2초.
