## 1. 안건 1 — 스테이지 구조: 디자이너 vs 밸런서 충돌

### 1-1. 입장 차이

| | game-designer (04) | game-balancer (balance-02) |
|---|---|---|
| 결론 | **2 챕터 × 12W + 체크포인트 W4/W8** 로 전면 개편 | **현행 6×5 유지 + 부분 조정** |
| 핵심 주장 | "5웨이브 강제 리셋이 카타르시스의 적" | "캐리오버로 빌드 보존됨, 톱니파가 도파민" |
| 근거 | `useGameState.jsx:601-602` 강제 와이프, T3 도달 직전 컷오프 | `CARRYOVER` (T2+ 5기, S2+ 3기), 스파이크 빈도 1위 |
| 리스크 | 망빌드 회복·세이브 호환·1런 25분 부담 | 후반 톱니파(91.5→73.7) 단조성, 도파민 부재 구간 |

### 1-2. 충돌 지점 분석
- 디자이너는 **빌드 깊이(T4 도달 시간)** 을 중시 — "T4를 못 보고 끝나는 5웨이브"
- 밸런서는 **도파민 빈도** 와 **회복감(파워 컴백)** 중시 — "S2W5 81HP → S3W1 40HP 의 학살감"
- **둘 다 데이터로 옳다**. 우선순위가 다른 것뿐.

### 1-3. 합의안 — **2단계 점진 개편**

전면 개편(디자이너) 도 현행 유지(밸런서) 도 아닌 **중간 경로**:

#### 1단계 (즉시) — 현행 6×5 골격 유지 + 캐리오버 강화 + 체크포인트 보상
밸런서 권고 채택. 디자이너의 비판 중 "초기화 비용" 만 골라 완화.
- **CARRYOVER 확장**: `maxTowers 5→8`, `minTier ≥2 → ≥1` (T1 도 1~2개 이월 허용)
  - 디자이너 비판 "T1 전부 손실" 직격
  - 밸런서 측 "캐리오버로 보존" 주장 강화
- **W3 = 미니 챌린지 웨이브** (`balance-01` 처방 B) — 단조성 1차 해소
- **스테이지 클리어 보상 차등화**: W5 보스 처치 시 캐리오버 추가 슬롯 +1 (퍼펙트 클리어 보너스)

#### 2단계 (검증 후) — 캠페인은 6×5 유지, **별도 모드 "롱런"** 으로 디자이너 안 도입
디자이너의 챕터-체크포인트 모델은 **신규 모드** 로 시즈하면 캠페인을 망치지 않고 실험 가능.
- 신규 모드: **"엔드리스 챕터" — 1런 = 2 챕터 × 12W + W4/W8 체크포인트**
- `domain/config/run-mode-constants.js` + `mode-abilities.js` 패턴 활용
- 캠페인 vs 엔드리스 챕터의 클리어율·평균 도달 웨이브를 비교 후 캠페인 본체에 적용 여부 결정

### 1-4. 왜 이 합의가 옳은가
- **블래스트 반경 최소**: 캠페인 본체는 마이너 패치(CARRYOVER + W3) 만, 큰 변경은 신규 모드에서 검증
- **두 입장 모두 만족**: 캐리오버 강화로 디자이너 핵심 비판 해결, 골격 유지로 밸런서 도파민 곡선 보존
- **롤백 가능**: 신규 모드는 별도 데이터 셋 → 실패해도 캠페인에 영향 없음

## 2. 안건 2 — 방어력/실드 도입

### 2-1. 디자이너 안 (05) 의 강점
- Armor=flat, Shield=별도 게이지로 **수학적으로 직교** — 튜닝 단순
- 광휘=관통, 전격=실드 브레이커로 **속성 카운터 메타 형성** (현재 6속성이 카운터 의미 없음을 해결)
- MVP/V2/V3 단계 분리 명확
- 처리 순서 명시: `damage → 속성보정 → Shield → Armor → HP`

### 2-2. 밸런서 관점 우려 (사용자 의도 반영)
- 안건 의도: "**현재 너무 쉬우니** 난이도 상승" → Armor 도입은 정확히 그 처방
- 단, `balance-01` 진단의 **초반 슬럼프(S1 W1~W3) + 후반 T4 병목** 곡선과 충돌 가능
  - S1 에 Armor 부여 시 → 더 쉬워지는 게 아니라 클리어 불가 위험
  - S6 에 Armor 부여 시 → 이미 빡빡한 곡선이 더 험해짐

### 2-3. 합의안 — **MVP 만 즉시 도입, 단 적용 범위·수치 더 보수적으로**

디자이너 안의 **MVP 단계만** 진행하되, 다음 보정:

| 항목 | 디자이너 원안 (05) | 합의 수정안 |
|---|---|---|
| 적용 시점 | elite/boss/suppressor 모두 동시 | **elite 만 먼저** (Stage 2 등장 시점부터) |
| Armor 수치 | elite 5 / suppressor 15 | elite **3** (T1=10dmg → 7 dmg, –30%) |
| 적용 스테이지 | 명시 X | **Stage 2 부터** (S1 은 무조건 armor=0, 진입 장벽 X) |
| 광휘 관통 | 50% | 50% 유지 (안건 강도 정합) |
| Shield / aegis | V2 로 분리 | V2 유지 ✅ |
| ArmorBreakEffect | V3 | V3 유지 ✅ |

근거:
- elite 만 먼저 = 노출 빈도 낮음 → 너프 / 추가 튠 여유 확보
- armor 3 = T1 절반이 아니라 30% 차감, T2(30) 10%, T3(100) 3% — 저티어 도태 압박 완만
- Stage 2 부터 = `balance-01` 의 초반 슬럼프 진단과 정합 (S1 은 그대로 둠)

### 2-4. 검증 KPI (도입 후 확인)
- **클리어율**: S2 첫 트라이 클리어율이 –10%p 이내 변동
- **속성 사용률**: 광휘 픽률이 도입 전 대비 +20%p 이내 (편향 방지)
- **체감**: "armor 보고 광휘 뽑고 싶다" 가 강제(–) 가 아닌 선호(+) 수준

## 4. 오픈 이슈

- **별점 시스템**: `StarRating.recordStage` 가 6스테이지 가정. 신규 모드 도입 시 별도 별점 테이블 필요.
- **세이브 호환**: 1단계는 호환 유지, 신규 모드는 신규 슬롯으로.
- **수치 미세 튜닝**: 합의 후 모든 수치는 `BalanceDesigner` 스킬 / `game-balancer` 에이전트로 핸드오프.

**핵심**: 두 에이전트의 결론이 충돌했지만, **"안건의 우선순위가 다르다"** 는 점을 인정하면 점진 개편이 답. 큰 변경(챕터 모델) 은 신규 모드에서 검증, 작은 변경(캐리오버·armor MVP) 은 캠페인에 즉시 적용.
