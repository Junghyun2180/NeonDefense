# [밸런스 02] 스테이지 구조 변경안 수치 검증

대상: 04-stage-structure-redesign.md (6×5 → 3×12 / 1×30 재편안)
HP: `30×(1+(s-1)×0.32)×(1+(w-1)×0.13)` (constants.js:205-212, lateBonus 제외)
골드: `15+4w+(w==5?15:0)` + `stageClearBonus=40+8s`

## 2. 결론 (3줄)

- **3×12가 곡선 매끄러움 1위**: 60→78→118→120, 단조 증가, 톱니 없음.
- **1×30은 경제 인플레 과다** (+81%, T4 W7 도달): 중반부터 도전감 소실.
- **현행 6×5는 톱니파**(91.5→73.7 후퇴 발생)지만 W5 스파이크 5회로 **도파민 빈도 1위**.

## 3. 반박: "스테이지 초기화는 빌드 만족감 저하"

수치 반박: **CARRYOVER**(constants.js:233-238)로 T2+ 5개, S2+ 3개 이월 → 빌드는 초기화되지 **않음**. 초기화는 적 HP뿐이며 톱니파가 **파워 회복감(도파민)**을 생성.

- 6×5에서 S2W5(81HP)→S3W1(40HP): 캐리오버 빌드가 1초 컷 가능 구간 부활 = "내 빌드가 강해졌다" 체감.
- 3×12/1×30: 이 회복 구간이 0~2회. 후반 12웨이브 단조 증가만 보임 → 빌드 노력 가시성 ↓.
- 스테이지 클리어 보상(영구 버프 3중1, lives+3) 6×5는 5회 / 1×30은 0회.

→ **스테이지 경계는 빌드를 깎지 않고, 오히려 만족감을 측정·증폭하는 구간.**

## 4. 권고

**현행 6×5 유지 + 부분 조정** (전면 개편 ❌)

근거: 캠페인 10-19분 모드에서는 곡선 매끄러움보다 **도파민 스파이크 빈도**가 우선. 캐리오버가 초기화 우려 이미 해소. 부분 조정은 lateWaveBonus 위치 / waveGrowth 미세 튜닝(별도 리포트 balance-01)으로 한정.
