# [제안 1] 원소 융합 타워 (Elemental Fusion) — T5 파라곤 시스템

## TL;DR
서로 다른 두 속성의 T4 타워 **2기를 합쳐** 전용 T5 "융합 타워" 1기를 만드는 End-Game 시스템.
6속성 × 6속성 = 15조합 고유 레전더리 타워로 후반 빌드 다양성과 수집 욕구를 동시에 잡는다.

## 문제 정의 (Why)
- 현재 최종 티어는 **T4 + 역할 선택(3종)** 에서 성장이 멈춘다. 스테이지 5~6 이후 플레이어는 "같은 T4를 더 뽑을 뿐" 새 의사결정이 없다.
- 서로 다른 속성 조합 보상이 없어 **"한 속성으로 올인"** 이 거의 항상 최적이다. 시너지 인센티브가 약하다.
- 캠페인 후반(S5~S6) / 런모드 후반(S4~S5) / 보스러시 고차 보스에서 **"더 강한 카드를 뽑는 재미"** 가 빠져 있다.
- 카드 수집 시스템(`collection-system.js`)은 있지만, **T4 역할 카드(18장)까지만** 존재 → 도감 성장도 T4에서 멈춤.

## 참고 사례 (역기획)

### Bloons TD 6 — Paragon
- 5티어(최상위) 타워 3개 + 다량의 재료로 **Paragon** 을 제작. 모든 경로의 장점을 합쳐 압도적 1기 타워로 변한다.
- 유저 반응: 엔드게임 목표가 뚜렷해져 플레이 시간이 크게 늘었다. 비용이 매우 비싸서 "모두가 항상 만들 수는 없다" 는 게 핵심.
- 우리와 공통점: 후반부 타워 풀이 닫혀 있는 문제 동일.
- 차이점: BTD6는 **단일 속성 3기**, 우리는 **이종 속성 2기** 를 쓴다. 조합 수로 다양성을 확보.

### Arknights — 모듈/특성화
- 오퍼레이터에 "모듈" 을 장착해 전혀 다른 역할로 바뀜. 같은 오퍼를 2번째로 키울 이유를 부여.
- 우리 맥락: **T4 역할 선택** 과 유사. 한 단계 더 나아가 **T5 융합** 으로 "키울 이유" 를 한층 더 쌓는다.

### Random Dice / Rush Royale — 합성 확장
- 같은 주사위 2개 합치면 상위 티어. 단, 합성 결과가 **때때로 다른 타입** 으로 나오는 변이 시스템이 있어 뽑기-조합 루프가 반복 소비된다.
- 우리는 **두 원소를 결정론적으로** 섞어 15개 고정 결과물로 만든다. (랜덤 변이 아님 → 빌드 설계 용이)

## 제안 기능 (What)

### 핵심 루프
1. T3 3개 조합 → T4 + 역할 선택 (**기존**)
2. **서로 다른 속성의 T4 타워 2기** + 골드 200G + "융합 코어" 1개 → T5 융합 타워 1기 (**신규**)
3. T5는 **맵에 1종당 최대 2기** 배치 가능 (스펙 폭증 방지)
4. 수집 도감에 `fusion-{A}-{B}` 카드 15장 추가 → 해금 욕구 자극

### 주요 규칙
1. **재료**: T4 속성 A 1기 + T4 속성 B 1기 (A ≠ B). 역할은 무관(어떤 역할 조합이든 OK).
2. **융합 코어**: 신규 재화. 보스 처치 시 10% 확률 드롭 + 스테이지 클리어 시 일정 확률로 획득 + 캠페인 S3 이상 스테이지 클리어 보상 1개 고정.
3. **T5 15종 정의**(가안, 아이콘/이름만 예시):
   - 🔥+❄️ `증기 폭발` — 광역 + 슬로우 디버프 전파
   - 🔥+⚡ `플라즈마 아크` — 화상 + 체인 (체인 적마다 화상)
   - 🔥+🌪️ `불꽃 폭풍` — 넓은 범위 화상 DoT + 넉백
   - 🔥+🌀 `소멸의 불꽃` — 화상 스택 최대치 돌파 허용
   - 🔥+💎 `성화 심판` — 처형 임계값 30% + 처형 시 폭발 화상
   - ❄️+⚡ `뇌우 결빙` — 빙결 중 적에게 전격 크리 +200%
   - ❄️+🌪️ `블리자드` — 광역 슬로우 + 주기적 넉백 펄스
   - ❄️+🌀 `절대 영도` — 관통 + 명중 적 슬로우 스택(최대 5)
   - ❄️+💎 `수정 첨탑` — 사거리 200 + 단일 고데미지 + 20% 확률 빙결
   - ⚡+🌪️ `정전기 소용돌이` — 원형 영역에 체인 지속 발동 (1초당 1회)
   - ⚡+🌀 `전자기 폭발` — 체인이 관통으로 확산 (체인 카운트×관통 카운트)
   - ⚡+💎 `뇌광 저격` — 선형 관통 + 관통 대상 전원 0.3초 스턴
   - 🌪️+🌀 `허무의 파열` — 적을 끌어당기고 20% 확률로 즉시 제거 (보스 제외)
   - 🌪️+💎 `광속 돌격` — 적 처치 시 재사용 대기시간 50% 감소 + 범위 넉백
   - 🌀+💎 `심판자의 눈` — 맵 전체 사정거리. 체력 20% 이하 적 전원에게 초당 저격
4. **스탯 베이스**: T4 × 1.8 damage / × 1.3 range / × 1.0 speed. 스킬 쿨다운 기반 고유 액티브 효과(선택사항) 추가 가능.
5. **제약**: 같은 융합 타워는 **맵당 최대 2기**. 인벤토리엔 무제한 소지.
6. **환급**: T5 판매 시 `towerBaseValues[4] × 2.5 × sellRefundRate = 540 × 2.5 × 0.5 = 675G`. (융합 코어는 환급 X)
7. **캐리오버**: T5는 캐리오버 후보에 **무조건 포함**(기존 `minTowerTier` 와 별개 규칙).

## 리스크 & 오픈 이슈

### 밸런스
- **스펙 폭증 위험**: T5 × 2기 = 현재 T4 × 4기 + α 수준이 되면 후반 스테이지가 쉬워짐.
  → `maxStage` 6 유지 + 스테이지 6 체력 스케일 재조정 필요 (**`BalanceDesigner` / `game-balancer` 에이전트 핸드오프**)
- **특정 조합 편향**: `심판자의 눈` 맵 전체 사거리가 지나치게 강력할 수 있음 → 쿨다운/동시 타격 수로 제한.

### 기존 시스템 충돌
- 서포트 타워 버프가 T5에 그대로 붙으면 곱셈 효과로 폭발 → **T5는 서포트 버프의 50%만 수령** 규칙 추가 검토.
- Prism 시스템과의 상호작용: "Prism T4 2기 → Prism T5" 결정론 보장 필요.

### 토큰/성능
- 15종 Ability 클래스는 500줄 규칙 위반 가능. `fusion-ability.js` 를 속성 짝별로 3~4개 파일로 분리할 것.
  (예: `fusion-ability-fire.js`, `fusion-ability-water.js` ...)

### UX 혼선
- "융합 조건(다른 속성 T4 2기)" 이 기존 "3개 조합" 규칙과 달라 플레이어 혼동 가능
  → 첫 T4 생성 시 튜토리얼 팝업 1회. 인벤토리에서 T4 2기 선택 시 "융합 가능!" 라벨 자동 표시.

### 오픈 이슈
- Q: 융합 시 재료 T4의 역할(A/B/C)을 결과에 반영할지? → **반영 안 함** (결정론 유지, 복잡도 관리) 제안
- Q: 런모드 웨이브 자동 시작 환경에서 융합 UI 열 시간이 있는가? → 웨이브 간 간격 2초는 짧음. 런모드는 **일시정지 허용** 또는 **인벤토리 상태에서 미리 조합** 으로 해결.
