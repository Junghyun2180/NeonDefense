# [제안 04] 스테이지 구조 재설계 — 6×5 모델 비판과 챕터-체크포인트 모델로의 전환

## TL;DR
**6스테이지×5웨이브 + 매 전환 시 보드 와이프**를 폐기, **2 챕터 × 12웨이브 + 체크포인트 W4/W8**(챕터 사이만 리셋)로 전환. 빌드 카타르시스·도파민 곡선·1런 시간(20~30분) 모두 개선.

---

## 1. 현행 구조 비판

### 1-1. 초기화의 비용
`useGameState.jsx:601-602`에서 스테이지 전환마다 `setTowers([])`/`setSupportTowers([])`로 보드를 강제로 비운다. 캐리오버는 `CARRYOVER.maxTowers=5 / maxSupports=3 / minTier≥2`(`constants.js:233-238`)로 매우 제한적 — T1·S1 전부 손실, 초과분도 손실. 1스테이지 끝에서 막 완성한 빌드를 보고 "다 부수네?" 하는 김빠짐이 **5웨이브마다 6번 반복**된다. TD 카타르시스(쌓은 진지가 적을 학살하는 누적)를 6번 끊는다.

### 1-2. 5웨이브 = 빌드 미숙성
스테이지 1 누적 보상 ≈ 195G + 클리어 48G = 243G. T4(누적 기댓값 ≈540G)는커녕 **T3도 빠듯**. 즉 빌드 폼이 잡히는 순간 = 보스 컷씬 = 와이프. 6스테이지 내내 **항상 초기 빌드만** 경험.

### 1-3. 반복 피로
동일 사이클(가챠→배치→W×5→보스→리셋→버프 1장)이 6회. 차이는 HP 계수·적 풀뿐, **구조의 새로움 없음**. Slay the Spire 3액트가 ‘일반/엘리트/보스’ 층위 변화를 주는 것과 대조.

### 1-4. 세이브 포인트 가치 (인정점)
초기화는 망빌드 수정 기회·버프 슬롯·환급 충전이기도 하다. 단, **체크포인트 가치와 보드 와이프는 분리 가능**한 별개 개념이다.

## 3. 추천안 — **C. 챕터-체크포인트**

### 구조
```
1 RUN = 2 챕터 × 12 W = 24 W
챕터 1: W1~W3 학습 / W4 체크포인트(보상·보드유지) / W5~W7 적풀 확장
       / W8 영구 버프 1장(보드유지) / W9~W11 풀 적풀 / W12 챕터 보스(캐리오버 + 와이프)
챕터 2: 동일, 보스는 다페이즈(제안 03 연동)
```

### 핵심 변경점
1. **와이프 1런당 1회**(챕터 사이만). 카타르시스 보존.
2. **체크포인트 W4/W8 = 보드 유지 + 보상만**.
3. **영구 버프 3회/런 유지** (W8×2 + 챕터1 보스). 인플레 차단.
4. **HP 곡선 재계수**: 챕터1 W12 ≈ 30·1.0·1.13¹¹·1.15 ≈ 132 HP, 챕터2 W12 ≈ 185 HP — 현행 S6W5(≈150 HP)와 정합.
5. **체크포인트 차등화**: W4=경제, W8=메타(버프), W12=빌드 재편(캐리오버+버프) → **다른 결정**으로 단조 해소.

### 왜 C인가
- A 대비: W4/W8 분기로 봉우리 추가, A는 평탄.
- B 대비: 챕터 와이프 1회로 빌드 재구성의 신선함 보존, B는 망빌드 회복 불가.
- D 대비: 버프 인플레 통제. D는 후반 OP 위험.

### 참고 사례
- **Slay the Spire (3 Acts)**: 액트 사이 카드/렐릭 유지, 층 보상(엘리트/휴식/상점)이 액트 내부 변주. → **체크포인트 차등화 차용**.
- **Kingdom Rush (스테이지당 12~25 W·단일 맵)**: 한 스테이지 내 빌드 깊이 + 영웅 누적. → **12W 채택 근거**.
- (보조) BTD6 라운드 80~100 단일 맵 = "리셋 없는 카타르시스" 검증 사례.

## 5. 리스크 / 오픈 이슈

1. **세이브 호환**: `stage:1~6` 스키마 마이그레이션 필요(stage→chapter+wave).
2. **별점 시스템**: `StarRating.recordStage`(useGameState:355)가 6스테이지 가정 → 챕터 단위로 재정의.
3. **인벤 30칸 포화**: 12W 가챠 누적 → W4 보상에 자동 분해 옵션 검토.
4. **W5~W11 단조 위험**: 제안 02(이벤트 웨이브) 동반 필수.
5. **신규 진입 부담**: 1런 25분 → 튜토리얼 챕터(6W) 별도 제공 검토.
6. **모드 호환**: 런/보스러시 `stageClearBonus`(`run-mode-constants.js:47,89`) 식 챕터 환원.

## 6. game-balancer 핸드오프 (수치 검증 필요)

1. **HP 곡선**: 챕터1 W12≈132, 챕터2 W12≈185 HP가 T4 1~2기 + T3 2~3기로 처치 가능 TTK.
2. **경제**: 24W 누적 보상 vs 현행 30사이클. T4 1기 도달 웨이브.
3. **체크포인트 보상 가중치**: W4 무료 가챠 1회 vs +60G 비교.
4. **영구 버프 가치 인플레**: 적용 구간 +50% 길어지므로 `damageUp +10%` 등 스택 캡 재검토.
5. **lives 정책**: 현행 `stageClearLivesRecover:3`을 챕터당 +6 vs 체크포인트당 +2 분배.
6. **별점 컷라인**: 챕터 단위 lives 손실 임계.
